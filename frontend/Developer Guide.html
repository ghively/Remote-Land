> DEVELOPER GUIDE — Architecture, Deployment & Extension Reference

Architecture

NAS_TERMINAL is a fully client-side single-page application. There is no build step — it runs directly in the browser via React 18 + Babel standalone. All state is managed in React; settings and webapp configs are persisted in localStorage.

The app is composed of independent JSX component files loaded via <script type="text/babel"> tags. Each file exports its components to window so they're accessible across file boundaries without a module bundler.

Design System

All colors, typography, spacing, and animation timing are defined as CSS custom properties in colors_and_type.css. The design language is Gene Hively's Cyber-Noir — green phosphor CRT terminals on a server-room wallpaper.

Rainbow Border

Window borders use a CSS conic-gradient on the ::before pseudo with a -webkit-mask XOR composite to reveal only the border area. The angle is driven by a JavaScript requestAnimationFrame loop that sets --gradient-angle on :root every frame, bypassing @property browser support requirements.

tmux as Terminal Backend RECOMMENDED

tmux is an excellent backend for NAS_TERMINAL. Unlike a raw shell, tmux provides persistent sessions (survive disconnects), window/pane multiplexing (map directly to our WM concept), and session sharing (multiple browser tabs see the same session).

Why tmux over a raw shell

  • Sessions survive network drops — reconnecting re-attaches to the same state
  • You can pre-create named sessions (nas-monitor, nas-logs, nas-admin) that the UI auto-attaches to
  • tmux windows ↔ terminal tabs; tmux panes ↔ split terminals — both concepts already exist in this WM
  • Scriptable: start long-running commands (htop, journalctl -f, docker stats) in named windows at boot

Setup with ttyd (tmux bridge)

# Install ttyd on Debian/Ubuntu
apt install ttyd

# Start ttyd bridging into a tmux session
tmux new-session -d -s nas-main   # Create persistent session
ttyd -p 7681 tmux attach -t nas-main

# Or start fresh tmux per connection (no persistence)
ttyd -p 7681 tmux new-session

# With auth (basic HTTP auth)
ttyd -p 7681 -c username:password tmux attach -t nas-main

# Run as systemd service
cat > /etc/systemd/system/ttyd.service <<EOF
[Unit]
Description=ttyd WebSocket terminal
After=network.target

[Service]
ExecStart=/usr/bin/ttyd -p 7681 tmux attach -t nas-main
Restart=always
User=root

[Install]
WantedBy=multi-user.target
EOF

systemctl enable --now ttyd

Pre-configure tmux sessions at boot

# /root/.config/nas-terminal/init-sessions.sh
#!/bin/bash

# Main admin session
tmux new-session -d -s nas-main -n admin

# System monitor window
tmux new-window -t nas-main -n monitor
tmux send-keys -t nas-main:monitor 'htop' Enter

# Log viewer window
tmux new-window -t nas-main -n logs
tmux send-keys -t nas-main:logs 'journalctl -f' Enter

# Docker stats window
tmux new-window -t nas-main -n docker
tmux send-keys -t nas-main:docker 'watch docker ps' Enter

# Start ttyd on each named window
ttyd -p 7681 tmux attach -t nas-main:admin   &
ttyd -p 7682 tmux attach -t nas-main:monitor &
ttyd -p 7683 tmux attach -t nas-main:logs    &

ttyd Configuration

ttyd exposes a WebSocket terminal server at ws://host:port/ws and serves an xterm.js frontend at http://host:port. NAS_TERMINAL replaces the xterm.js frontend while reusing the WebSocket protocol.

xterm.js Integration

To replace the simulated shell in TerminalPane.jsx with a real WebSocket terminal, follow these steps:

1. Add xterm.js to the HTML

<!-- In NAS Terminal.html, add to <head>: -->
<link  rel="stylesheet" href="https://cdn.jsdelivr.net/npm/xterm@5.3.0/css/xterm.css">
<script src="https://cdn.jsdelivr.net/npm/xterm@5.3.0/lib/xterm.js"></script>
<script src="https://cdn.jsdelivr.net/npm/xterm-addon-fit@0.8.0/lib/xterm-addon-fit.js"></script>
<script src="https://cdn.jsdelivr.net/npm/xterm-addon-web-links@0.9.0/lib/xterm-addon-web-links.js"></script>

2. Replace TermSession in TerminalPane.jsx

function TermSession({ session, ttydUrl }) {
  const containerRef = useRef(null);
  const termRef = useRef(null);
  const wsRef = useRef(null);

  useEffect(() => {
    if (!containerRef.current) return;

    // Init xterm
    const term = new Terminal({
      fontFamily: 'JetBrains Mono, Courier New, monospace',
      fontSize: 14,
      theme: {
        background: '#000800',
        foreground: '#ccffcc',
        cursor:     '#00ff00',
        green:      '#00ff00',
        cyan:       '#00f3ff',
        magenta:    '#8000ff',
      },
      cursorBlink: true,
    });

    const fitAddon = new FitAddon.FitAddon();
    term.loadAddon(fitAddon);
    term.open(containerRef.current);
    fitAddon.fit();
    termRef.current = term;

    // Connect to ttyd WebSocket
    const ws = new WebSocket(ttydUrl || `ws://${session.host}:7681/ws`);
    ws.binaryType = 'arraybuffer';
    wsRef.current = ws;

    ws.onopen = () => {
      // ttyd protocol: send auth token first
      ws.send(JSON.stringify({ AuthToken: '' }));
    };

    ws.onmessage = (event) => {
      if (typeof event.data === 'string') {
        const msg = JSON.parse(event.data);
        if (msg.Arguments) term.write(atob(msg.Arguments));
      } else {
        const data = new Uint8Array(event.data);
        if (data[0] === 1) term.write(data.slice(1));
      }
    };

    term.onData(data => {
      if (ws.readyState === WebSocket.OPEN) {
        ws.send('0' + data);
      }
    });

    // Resize handler
    const resize = () => {
      fitAddon.fit();
      if (ws.readyState === WebSocket.OPEN) {
        ws.send(JSON.stringify({
          columns: term.cols, rows: term.rows
        }));
      }
    };
    const ro = new ResizeObserver(resize);
    ro.observe(containerRef.current);

    return () => {
      ro.disconnect();
      ws.close();
      term.dispose();
    };
  }, [session.id, ttydUrl]);

  return (
    <div ref={containerRef}
      style={{ flex: 1, overflow: 'hidden', background: '#000800', padding: '4px' }}
    />
  );
}

ttyd protocol: ttyd uses a simple framing protocol. Byte 0 prefix = stdin data; byte 1 prefix = stdout/stderr from server. The initial handshake sends {"AuthToken": ""}.

SSH Proxy Setup

For connecting to remote machines from your NAS (not just the NAS itself), run a WebSocket SSH proxy on the NAS:

Option A: webssh (Python)

# Install
pip install webssh

# Start
wssh --port=8888 --address=0.0.0.0

# Connects to: ws://nas.local:8888/?hostname=HOST&username=USER

Option B: ssh2ws (Node)

# Install
npm install -g ssh2ws

# Start
ssh2ws --port 8888

# Then in TerminalPane.jsx, connect via:
const ws = new WebSocket(
  `ws://nas.local:8888/?host=${targetHost}&user=${user}&pass=${pass}`
);

Option C: Cloudflare Tunnel (remote access)

# Install cloudflared on NAS
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
chmod +x cloudflared-linux-amd64
mv cloudflared-linux-amd64 /usr/local/bin/cloudflared

# Expose NAS_TERMINAL securely without port forwarding
cloudflared tunnel --url http://localhost:8765

Window Manager Overview

The WM is implemented in WindowManager.jsx and orchestrated in NASTerminal.jsx (WMDesktop component). Two modes:

  • Tiled mode — dwindle recursive bisect. Windows are positioned by the algorithm; drag is disabled. Resize via tiling handles.
  • Float mode — drag via titlebar, resize via SE corner + right/bottom edge handles. Position stored in window state.

Toggle with [TILED]/[FLOAT] in the dock or Super+T.

Dwindle Tiling Algorithm

The algorithm recursively bisects a rectangle, alternating horizontal/vertical splits based on current aspect ratio — wider rects split horizontally, taller ones vertically.

function dwindleLayout(count, rect, depth = 0) {
  if (count === 0) return [];
  if (count === 1) return [{ ...rect }];
  const { x, y, w, h } = rect;
  const gap = 8;
  const splitHoriz = w >= h;
  // Split into A (first window) and B (recurse remaining)
  return [A, ...dwindleLayout(count - 1, B, depth + 1)];
}

With 6 windows the layout looks like: 1 large left, then the right half keeps recursing vertically. Transitions use cubic-bezier(0.25,1,0.5,1) at 350ms — Hyprland's ease-out-quart.

Nginx Configuration

A recommended nginx config that serves NAS_TERMINAL, proxies the API services, adds CORS headers for iframes, and upgrades WebSocket connections for ttyd:

server {
    listen 80;
    server_name nas.local;

    # NAS Terminal static files
    location / {
        root /opt/nas-terminal;
        index "NAS Terminal.html";
        try_files $uri $uri/ /NAS Terminal.html;
    }

    # ttyd WebSocket terminal
    location /terminal/ {
        proxy_pass http://127.0.0.1:7681/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 43200;
    }

    # Radarr — strip X-Frame-Options so iframe works
    location /radarr/ {
        proxy_pass http://127.0.0.1:7878/radarr/;
        proxy_hide_header X-Frame-Options;
        add_header X-Frame-Options "SAMEORIGIN" always;
        proxy_set_header Host $host;
    }

    # Sonarr
    location /sonarr/ {
        proxy_pass http://127.0.0.1:8989/sonarr/;
        proxy_hide_header X-Frame-Options;
        add_header X-Frame-Options "SAMEORIGIN" always;
    }

    # SABnzbd — add CORS for API calls
    location /sabnzbd/ {
        proxy_pass http://127.0.0.1:8080/sabnzbd/;
        proxy_hide_header X-Frame-Options;
        add_header Access-Control-Allow-Origin "*" always;
        add_header Access-Control-Allow-Methods "GET, POST" always;
    }

    # Emby / Jellyfin
    location /emby/ {
        proxy_pass http://127.0.0.1:8096/;
        proxy_hide_header X-Frame-Options;
        add_header X-Frame-Options "SAMEORIGIN" always;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

Security Considerations

NAS_TERMINAL runs as root by default in demo mode. Before exposing it outside your LAN, implement proper authentication.

Authentication options

  • Nginx basic auth — Quick: htpasswd -c /etc/nginx/.htpasswd gene then add auth_basic to nginx config
  • Authelia / Authentik — Full SSO with 2FA, forward-auth proxy
  • Cloudflare Access — Zero-trust with email OTP, no port forwarding needed
  • WireGuard VPN — Only expose on VPN interface; access from anywhere securely

Replace the demo login

// In NASTerminal.jsx — handleLogin function, replace VALID_CREDS check with:
const handleLogin = async (user, pass, host) => {
  const res = await fetch('/api/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ user, pass })
  });
  if (res.ok) {
    const { token } = await res.json();
    sessionStorage.setItem('nas_auth', JSON.stringify({ user, host, token }));
    setAuth({ user, host });
  } else {
    setError('> AUTH FAILED');
  }
};

API key security

API keys are stored in localStorage. For a shared device, consider storing them server-side and proxying API calls through a backend that injects the key, so keys are never visible in the browser.

Adding Custom System Panels

System panels (file manager, sysmon, etc.) live in SystemPanels.jsx. To add a new one:

  1. Create the React component: function MyPanel() { ... }
  2. Export: window.MyPanel = MyPanel;
  3. Add to WIN_DEFAULTS in NASTerminal.jsx
  4. Add a case 'mypanel': in renderWindowContent
  5. Add to PANEL_APPS in AppLauncher.jsx

Adding Webapps

Webapps open in an iframe window. Add them via the in-app launcher ([+ ADD WEBAPP] at the bottom of the launcher overlay) or directly in AppLauncher.jsx:

// In AppLauncher.jsx — DEFAULT_APPS array:
{
  id: 'grafana',
  name: 'Grafana',
  desc: 'Metrics Dashboard',
  url: 'http://nas.local:3000',
  icon: '[GR]',
  cat: 'ADMIN'
}

iframe blocking: Many services set X-Frame-Options: DENY. The [↗] button in every webapp window opens the URL in a real browser tab as a fallback. Configure nginx to strip and reset X-Frame-Options for services you want to embed.
